## Auth token format
Tokens are HMAC-SHA256 signed: `base64url(payload).base64url(sig)`. Generated by `generateToken()` in `artifacts/api-server/src/lib/auth.ts`. Verified in `parseToken()` with timing-safe comparison.

**Why:** The original implementation used unsigned base64 JSON — any client could forge tokens. The HMAC approach uses SESSION_SECRET for integrity.

**How to apply:** Always call `generateToken(userId, role)` on login/register. Never issue plain base64 tokens. SESSION_SECRET env var is required in prod (falls back to a long dev string).

## Password hashing
Uses SHA-256 + SESSION_SECRET as pepper. This is dev-appropriate only.

**Why:** bcrypt not installed; sha256 is good enough for a prototype.
**How to apply:** For production, install bcrypt and replace `hashPassword()`. Do not change the pepper without re-hashing all passwords.

## Frontend auth wiring
`setAuthTokenGetter(() => localStorage.getItem('jwtToken'))` is called at module level in `App.tsx`, before `QueryClientProvider` mounts.

**Why:** The custom-fetch layer only attaches Authorization headers when a getter is registered. Without this call, all protected hooks sent no token.
